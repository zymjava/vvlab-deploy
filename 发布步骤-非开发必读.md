# 发布新版本步骤（非开发也能照着做）

本文档说明：**当简历或页面有更新时，怎样把新内容发布到网站上**。不需要会写代码，只要按步骤操作即可。  
以 **zxy 个人简介站（http://zxy.vvlab.xyz）** 为例，其他站点（zym、photo）思路相同。

---

## 一、你需要准备的东西

- 能打开 **vvlab/zxy** 这个文件夹的电脑（例如用 Cursor、VS Code 或记事本都行）。
- **GitHub** 账号，并且 zxy 项目已经关联到 GitHub（zymjava/zxy-space）。
- 能 **SSH 登录到服务器** 139.224.31.98（或有人帮你执行服务器上的命令）。
- （可选）**阿里云** 账号，用于查看镜像是否构建成功。

---

## 二、发布流程总览（5 步）

```
1. 在电脑上改内容（文案/照片/样式）
      ↓
2. 用 Git 提交并推送到 GitHub
      ↓
3. 在阿里云 ACR 触发构建（或等自动构建）
      ↓
4. 在服务器上执行一条“重启”命令
      ↓
5. 浏览器打开网站检查是否更新成功
```

下面每一步都写了具体操作，**完全按顺序做**即可。

---

## 三、详细步骤

### 第 1 步：在电脑上修改内容

1. 打开 **vvlab** 文件夹，进入 **zxy** 子文件夹。
2. 根据你要改的内容，打开对应文件：
   - **只改文字（姓名、电话、工作经历等）**：用记事本或编辑器打开 **data/content.json**，按里面的格式改，改完保存。
   - **要换头像或照片**：把新图片放到 **assets/photos/** 文件夹里（例如改名为 avatar.png），然后打开 **data/photos.json**，确认路径写的是 `assets/photos/avatar.png`（或你用的文件名）；再打开 **data/content.json**，在「基本信息」里 `"photo": "avatar"` 保持为 avatar（或和你 photos.json 里写的名字一致）。
   - **要改页面样式（颜色、排版等）**：在 **css** 或 **js** 文件夹里改对应文件，改完保存。
3. 确认所有改过的文件都已**保存**。

---

### 第 2 步：提交并推送到 GitHub

这一步是把“你电脑上的修改”同步到 GitHub 上，这样阿里云才能用最新代码构建镜像。

1. 打开 **命令行**（Windows 可在文件夹里按住 Shift 点右键，选“在此处打开 PowerShell 窗口”；或打开 Cursor/VS Code 的终端）。
2. 进入 zxy 目录（根据你的实际路径改）：
   ```bash
   cd c:\Users\admin\vvlab\zxy
   ```
3. 依次执行下面三条命令（每行回车一次）：
   ```bash
   git add .
   git commit -m "更新简历内容"
   git push origin main
   ```
   - 如果提示要登录 GitHub，按提示在浏览器里登录或输入账号密码/Token。
   - 如果分支不叫 main 而叫 master，把最后一条里的 `main` 改成 `master`：  
     `git push origin master`
4. 看到类似 “Everything up-to-date” 或 “Writing objects: 100%” 就表示推送成功。可以打开 GitHub 网站，进入 **zymjava/zxy-space** 仓库，确认刚改的文件已经是最新的。

---

### 第 3 步：在阿里云 ACR 触发镜像构建

阿里云会用 GitHub 上最新代码“打一个镜像包”，供服务器拉取。

1. 打开浏览器，登录 **阿里云**，进入 **容器镜像服务**（在控制台里搜“容器镜像”即可）。
2. 选择 **个人版**，找到镜像仓库 **zxy-repo**，点进去。
3. 点 **构建** 或 **构建记录**，再点 **立即构建**（或“触发构建”）。
4. 等待构建完成（通常一两分钟）。构建状态变成 **成功** 即可。
   - 若构建失败，看构建日志里报错；常见情况是“拉取 nginx 超时”，这时需要开发把 Dockerfile 里的基础镜像改成国内地址（详见 deploy/TROUBLESHOOTING.md 问题四）。

---

### 第 4 步：在服务器上“重启”站点（让新镜像生效）

服务器需要重新拉取最新镜像并重启一次，网站才会显示新内容。

1. 用 **SSH** 登录到服务器 **139.224.31.98**（使用 root 或已有账号）。
2. 登录成功后，在终端里**一条一条**执行下面两条命令：
   ```bash
   kubectl rollout restart deployment/zxy-site -n demo
   kubectl rollout status deployment/zxy-site -n demo --timeout=120s
   ```
3. 第二条命令会等一会儿，最后出现 `deployment "zxy-site" successfully rolled out` 就表示重启成功。若一直卡住或报错，可联系开发看 deploy/TROUBLESHOOTING.md。

---

### 第 5 步：在浏览器里检查网站是否更新

1. 打开浏览器，访问：**http://zxy.vvlab.xyz/**（注意是 **http**，不要用 https）。
2. 刷新页面（Ctrl+F5 或 Cmd+Shift+R 强制刷新），看文案、照片、样式是否已经是你刚改的内容。
3. 若没更新：等 1～2 分钟再刷新；若仍不对，检查第 3 步构建是否成功、第 4 步两条命令是否都执行成功。

## 四、常见问题（非开发向）

- **“git 不是内部或外部命令”**  
  说明本机没装 Git。需要先安装 Git（从 https://git-scm.com/ 下载），安装后再做第 2 步。

- **“push 的时候要账号密码”**  
  GitHub 现在一般要用 **Token** 当密码。在 GitHub 网站 → 设置 → Developer settings → Personal access tokens 里生成一个 Token，在提示输入密码时粘贴这个 Token。

- **“服务器我不知道怎么 SSH”**  
  请让管理员帮你登录一次，或把第 4 步的两条命令发给管理员，让他在服务器上执行。

- **“访问 zxy.vvlab.xyz 显示 502 或打不开”**  
  可能是服务器上 Nginx 端口配置错了，或 Pod 没起来。详细排查步骤见 **deploy/TROUBLESHOOTING.md**，可把该文档发给开发或运维对照处理。

- **“我只改文案，也要做第 3 步构建吗？”**  
  要。只要希望网站显示新内容，就必须重新构建镜像（第 3 步），再在服务器上重启（第 4 步）。

## 五、小结：下次发布时你要做的

| 步骤 | 你在哪里做 | 做什么 |
|------|------------|--------|
| 1 | 自己电脑 vvlab/zxy | 改 data/content.json 或照片、样式，保存 |
| 2 | 自己电脑，命令行 | `cd zxy` → `git add .` → `git commit -m "更新简历"` → `git push origin main` |
| 3 | 阿里云 ACR 控制台 | 打开 zxy-repo → 构建 → 立即构建，等成功 |
| 4 | 服务器 139.224.31.98（SSH） | 执行 `kubectl rollout restart deployment/zxy-site -n demo` 和 `kubectl rollout status ...` |
| 5 | 浏览器 | 打开 http://zxy.vvlab.xyz 刷新，确认已更新 |

更细的排障（502、镜像拉取失败、构建超时等）见 **deploy/TROUBLESHOOTING.md**。
